i
Keyboard switching: The numpad is the default input when the sheet opens. Tapping the title field swaps the numpad for the standard iOS keyboard. Tapping the amount area or anywhere else swaps back to the numpad. The bottom area toggles fluidly between the two — same pattern as Copilot Money and Revolut.
ii
Batch entry: After saving, the sheet plays a brief checkmark animation (200ms), then resets all fields while staying open. The user can immediately enter another transaction or swipe down to dismiss. This enables rapid-fire expense logging.
iii
Smart defaults applied on open: Date → Today. Account → last used. Type → Expense (85%+ of manual entries). This means a minimal entry is just: type amount (3-5 numpad taps) + tap Category pill + pick from sheet (1 tap) + save (1 tap) = 6-8 taps total.
Design Rationale
Amount First

The amount is the hero

Every top finance app (Copilot, Revolut, YNAB, Monarch) puts the amount as the largest, most prominent element. At 40pt bold with monospaced digits, it dominates the visual hierarchy and gives users confidence in what they're logging.

Custom Numpad

Purpose-built for money

7 of 9 top finance apps use a custom numpad instead of the system keyboard for amounts. Larger tap targets, no mode-switching, and room for domain-specific keys. The numpad is the default; the standard keyboard appears only when typing the title.

Unified Pill Row

All metadata in one scrollable strip

Date, Category, Account, Note, and Tags live in a single horizontally scrollable pill row. Filled pills show the value with a colored dot; empty pills show a dashed outline. Tapping any pill opens its picker sheet. Keeps the layout minimal and scannable.

Smart Defaults

Two fields pre-filled on open

Date defaults to today (80%+ of entries are for today). Account defaults to last-used. This eliminates two mandatory fields from every entry, reducing the minimum interaction to: amount + category + save.

Explicit Routing

Users always know where it goes

The save button clearly shows "Save to Ledger" (all fields complete) or "Save as Draft → Inbox" (missing required fields). No ambiguity about the destination. The inbox/ledger model remains — just surfaced more transparently.

Progressive Disclosure

Required fields visible, optional on demand

Amount, title, and categories are always visible. Date and account are shown as filled pills (smart defaults). Notes and hashtags appear as ghost pills — tappable but not demanding attention. Nothing scrolls; everything fits in one view.
